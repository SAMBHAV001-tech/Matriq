# Matriq - Smart Inventory Analytics System

A production-style FastAPI backend using Supabase PostgreSQL for a SAP-inspired custom ABAP ALV report scenario.

## Project Overview

Matriq serves as an enterprise inventory monitoring mock system, providing detailed insights into material supply, vendor metrics, and stock alerts. It utilizes clean analytical layers to evaluate dynamic risk scores and compute reorder recommendations.

## Setup Instructions

### 1. Create a Supabase Project
1. Log into Supabase and create a new project.
2. Navigate to Database -> Database Settings.
3. Scroll down to Connection String and select **URI**. Check “Use Connection Pooling”.

### 2. Configure Environment
Create a `.env` file in the `backend/` root:
```env
DATABASE_URL=postgresql+psycopg://YOUR_USER:YOUR_PASSWORD@YOUR_HOST:5432/postgres
PORT=8000
```
*(Ensure to replace `postgresql://` generated by Supabase with `postgresql+psycopg://` for SQLAlchemy).*

### 3. Local Run & Deployment
```bash
python -m venv venv
# Activate venv
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

### 4. Seed Data
Trigger a `POST` request to `http://localhost:8000/seed` once to automatically populate tables with initial realistic materials (Steel Rods, Lubricants, etc), mock Vendors, and Stock instances!

### 5. Swagger Interaction
View the visually categorized interactive docs at `http://localhost:8000/docs`.

### Deployment Notes
This backend is fully dockarized `Dockerfile`. Deploy easily onto Render or Railway. Make sure to define the `DATABASE_URL` environment variables on the hosting platform.
